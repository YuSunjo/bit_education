	1. Thread
	가) Time Slicing Multitasking
	 - 아주 짧은 단위 시간동안 운영체제의 일부와 조합된 특정 코드가 시스템을 장악
	 - 짧은 시간동안 작업의 전환이 발생
	 - 비 선점형(cooperative) 멀티태스킹 : 운영체제가 강제로 작업 제어권을 획득할 수 없음
	 - 선점형(preemtive) 멀티태스킹 : 운영체제에 의해 수행, 운영체제가 시스템 자원배분과 프로세스 실행결정
	
	 나) 스케쥴링
	  - 윈도우에서는 시분할 멀티태스킹에서 일정시간 동안 특정 프로그램을 수행하는 단위를 퀀텀(Quantum) 이라고 함
	  - 퀀텀은 근본적으로 20ms 임
	  - 각 쓰레드는 우선순위(priority)를 가짐
	   - 윈도우는 스케줄링 퀀텀이 쓰레드에게 배분
	   - 윈도우는 스케줄 기본단위가 쓰레드이며, 유닉스는 프로세스 단위
	   - 윈도우는 모든 프로세스가 기본적으로 한개 이상의 쓰레드를 가짐
	
	
	다) 프로세스
	 - 어플리케이션에 리소스를 관리하는 가장 큰 체계
	 - 프로세스는 일정 고유영역의 메모리를 할당받음
	 - 프로세스 상호간 통신을 위한 방법이 IPC(InterProcess Call)
	 
	
	 라) 쓰레드
	  - 프로세스내의 실행코드
	  - 윈도우는 프로세스당 최소 하나의 쓰레드가 동작
	  - 기본적으로 발생되는 쓰레드를 Primary Thread 라 함(인스턴스 넘버 0)
	 
	
	 마) 컨텍스트
	  - 운영중인 시스템에서 쓰레드가 동작하기 위한 정보
	  - 실행중인 쓰레드가 운영체제에 의해 강제로 CPU를 빼앗길 때, 다음 복귀 시 계속 작업을 수행할 수 있도록 현재 상태를 그대로 저장한 내용
	  - 레지스터내용, 스택포인트, 쓰레드 주소공간, 운영 변수 등
	  - 시분할 멀티태스킹에서 실행 쓰레드가 바뀔 대 컨택스트 스위칭이 일어남
	  - Win32 구조체 중 유일하게 CPU 종속적인 부분
	  -실행중인 쓰레드가 운영체제에 의해 강제로 cpu를 빼앗길 때, 다음 복귀 시 계속 작업을 수행할 수 있도록 
		현재 상태를 그대로 저장한 내용 
	  -실행 쓰레드가 바뀔 때 마다 컨텍스트 스위칭이 일어남 
 	  -쓰레드가 많을 경우 컨텍스트 스위칭 때문에 느려질수 있음 
	  -최적의 쓰레드 갯수 -> 코어 갯수*2
